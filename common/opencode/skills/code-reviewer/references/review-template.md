# Review Template

Use this structure for review output. Keep it concise for small diffs.

## Summary
- One to three sentences on intent and overall risk level.

## Blockers
- None, or list each finding with file path and rationale.

## Major findings
- List each issue with file path, impact, and suggested fix.

## Minor findings
- List each improvement or risk with brief rationale.

## Nits
- Optional style or clarity suggestions.

## Questions
- Clarifications needed to validate assumptions.

## Tests
- Note existing tests and gaps; propose specific cases.

## Verification
- Manual checks or commands to validate behavior, if relevant.
